Novedades del controlador JDBC

Controlador Microsoft JDBC 4.0 para SQL Server incluye varias características nuevas y mejoras.

Información sobre la conexión con una base de datos de SQL Azure

Ahora hay un tema que contiene información sobre la conexión a una base de datos de SQL Azure. Vea Conectar con una base de datos en SQL Azure para obtener más información.

Compatibilidad con alta disponibilidad y recuperación ante desastres

El Controlador Microsoft JDBC 4.0 para SQL Server admite conexiones de alta disponibilidad y recuperación ante desastres con grupos de disponibilidad AlwaysOn en SQL Server 2012. Vea Compatibilidad del controlador JDBC con alta disponibilidad y recuperación ante desastres para obtener más información.

Usar la autenticación integrada de Kerberos para conectar con SQL Server

El Controlador Microsoft JDBC 4.0 para SQL Server cuenta con autenticación integrada de Kerberos de tipo 4 para conectarse a una base de datos de SQL Server. Para obtener más información, vea Usar la autenticación integrada de Kerberos para conectar con SQL Server. (La autenticación integrada de Kerberos de tipo 2 está disponible en las versiones anteriores a la 4.0 del Controlador Microsoft JDBC para SQL Server.)

Tener acceso a información de diagnóstico en el registro de eventos extendidos

Puede obtener acceso a información del registro de eventos extendidos del servidor para entender los errores de conexión. Para obtener más información, vea Tener acceso a información de diagnóstico en el registro de eventos extendidos.

Compatibilidad adicional para las columnas dispersas

Si su aplicación ya accede a los datos de una tabla que usa columnas dispersas, debe observar un incremento del rendimiento. Puede obtener información acerca de las columnas (incluida la información de las columnas dispersas) con Método getColumns (SQLServerDatabaseMetaData). Para obtener más información acerca de las columnas dispersas de SQL Server, vea Usar columnas dispersas.

El método isSparseColumnSet se agrega a la clase SQLServerResultSetMetaData para indicar si una columna de un conjunto de resultados es un conjunto de columnas dispersas.

Xid.getFormatId

Antes de Controlador Microsoft JDBC 4.0 para SQL Server, el controlador JDBC recibía un identificador de formato de la aplicación pero no pasaba el valor al servidor de base de datos. En su lugar, el controlador JDBC pasaba un identificador de formato de 0 al servidor de base de datos.

A partir de Controlador Microsoft JDBC 4.0 para SQL Server, el controlador JDBC pasará el identificador de formato de la aplicación al servidor de base de datos. Para obtener el comportamiento actualizado, asegúrese de que se ha actualizado sqljdbc_xa.dll en el servidor. Para obtener más información acerca de cómo copiar una versión actualizada de sqljdbc_xa.dll en el servidor, vea Descripción de las transacciones XA.

No se admite Itanium para aplicaciones del controlador JDBC 4.0

Las aplicaciones del Controlador Microsoft JDBC 4.0 para SQL Server no se pueden ejecutar en un equipo basado en Itanium.
