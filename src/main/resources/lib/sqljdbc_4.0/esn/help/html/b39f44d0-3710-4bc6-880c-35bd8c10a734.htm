Trabajar con tipos de datos (JDBC)

La función principal del Controlador Microsoft JDBC para SQL Server es permitir a los desarrolladores de Java el acceso a datos contenidos en bases de datos de SQL Server. Para ello, el controlador JDBC realiza la conversión entre los tipos de datos de SQL Server y los tipos y objetos en lenguaje Java.

Para obtener una descripción detallada de los tipos de datos de SQL Server y el controlador JDBC, incluidas sus diferencias y el modo de conversión a los tipos de datos en lenguaje Java, consulte Describir los tipos de datos del controlador JDBC.

Para trabajar con los tipos de datos de SQL Server, el controlador JDBC proporciona los métodos get<Type> y set<Type> para las clases SQLServerPreparedStatement y SQLServerCallableStatement, y proporciona los métodos get<Type> y update<Type> para la clase SQLServerResultSet. El método usado depende del tipo de datos con que se trabaja y del uso de conjuntos de resultados o consultas.

En los temas de esta sección se describe cómo usar los tipos de datos del controlador JDBC para obtener acceso a los datos de SQL Server en las aplicaciones de Java.
